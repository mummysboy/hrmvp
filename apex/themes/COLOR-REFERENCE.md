## üé® Default (Lavender) ‚Äî Original Theme

### Primary Colors
- **Accent Primary** `#E9B0FF` ‚Äî Buttons, links, primary actions
- **Accent Secondary** `#A3D6FF` ‚Äî Focus states, secondary elements
- **Accent Tertiary** `#FFD8A8` ‚Äî Badge accents

### Status Colors
- **Success** `#6DD58C` ‚Äî Approved, active, confirmed
- **Warning** `#FFD480` ‚Äî Pending, caution, needs attention
- **Danger** `#FF7A7A` ‚Äî Rejected, error, critical
- **Info** `#80D4FF` ‚Äî Information, notifications

### Background Colors
- **Primary BG** `#FAFAFA` ‚Äî Main page background
- **Secondary BG** `#F5F5F5` ‚Äî Cards, containers
- **Tertiary BG** `#EFEFEF` ‚Äî Hover states, disabled

### Text Colors
- **Primary Text** `#1E1E1E` ‚Äî Headlines, main text
- **Secondary Text** `#5A5A5A` ‚Äî Descriptions, labels
- **Tertiary Text** `#888888` ‚Äî Helper text, hints

## üîß Using Colors in Custom CSS

To use theme colors in your own components:

```css
/* Use CSS variables to stay consistent with theme */
.my-component {
  background-color: var(--color-bg-secondary);
  color: var(--color-text-primary);
  border: 1px solid var(--color-border);
  accent-color: var(--color-accent-primary);
}

.my-button {
  background: linear-gradient(
    145deg,
    var(--color-accent-primary),
    var(--color-accent-secondary)
  );
  color: #FFFFFF;
}
```

## üé® Theme Color Harmony

Each theme uses complementary color theory to ensure:
- ‚úÖ Primary and secondary colors work together
- ‚úÖ Status colors (success/warning/danger) are distinct
- ‚úÖ Good contrast between text and backgrounds (WCAG AA)
- ‚úÖ Professional appearance across all contexts
